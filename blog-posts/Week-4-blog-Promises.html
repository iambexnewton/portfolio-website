What we covered this week…

TUESDAY: Asynchronus code.

Javascript is a single threaded programming language, the javascript engine can only process one thing at a time, reading from top to bottom. However, there is magic inside Javascript. Asynchronous javascript code runs from top to bottom also, but when is sees an asynchronous function or piece of code it will take that to one side and perform the asynchronous task and the synchronous code without blocking the code thread from completing.

To explore this asynchronous javascript we used the node application protocol interface (api) setTimeout() function to implement a time delay. When javascript reads this node api setTimeout() it processes it separately, through the api event loop and waits for the call stack to become empty before running to completion, thus enabling javascript to run two functions synchronously, without blocking the code.

WEDNESDAY: Asynchronus code.

Today we explored asynchronous code by writing asynchronous functions. We explored this by using callbacks.

Where nodes api lacks, callback prospered. nodes api appeared to me like it could become rather messy and difficult to read your code if you were trying to perform multiple functions, as the order they are returned may not be exactly guaranteed. The time that is passed as a parameter in setTimeout() is the minimum time for the delay, not the time it will actually take.

Callbacks are a way to make sure certain code doesn’t execute until other code has finished.

THURSDAY: The Internet.

How does the internet actually work? How does information move around the internet?

I found this really interesting. It’s something I use everyday but never really delved into it. Let’s take a brief look…

You’re scrolling through cat pictures on your phone, how do the pictures get to your phone?

The picture is hosted on a webserver. Your phone sends a request to the webserver for the image.

The request is sent in a ‘packet’. Routers will then direct the packet sent by your phone to the web server. The web server then reads the request held in the packet. Images can often be very large so they can be split into many packets, like a jigsaw puzzle.

Routers then send your Image packets back and put back together, each packet will have instructions on where it belongs, some may take longer than others to arrive back. When all the packets are back, the image will be displayed.

FRIDAY: Promises.

A promise in javascript is “a proxy for a value not necessarily known when the promise is created.” — MND Web Docs

A promise is an object that can be returned synchronously from an asynchronous function.

A promise can only have three states…

Pending: Hasn’t fulfilled or rejected the promise yet.

Fulfilled: The action relating to the promises succeeded

Rejected: The action was rejected.
